## 🔍 Overview

**Focus Mentor** is a next-generation educational platform that combines the power of artificial intelligence with collaborative learning tools. Designed for students, educators, and learning enthusiasts, it provides a comprehensive suite of features to enhance productivity, facilitate knowledge sharing, and create an optimal learning environment.

### 🎯 Mission
To democratize access to personalized, AI-enhanced learning experiences while fostering collaboration and knowledge sharing in educational communities.

### 🌟 Key Highlights
- **AI-Powered Study Planning** - Personalized learning paths with intelligent recommendations
- **Virtual Meeting Rooms** - WebRTC-based video conferencing with AI meeting summaries
- **Interactive PDF Chat** - Conversational document analysis with context-aware responses  
- **Smart Resource Curation** - AI-driven content discovery and organization
- **Productivity Tools** - Pomodoro timer, note-taking, and progress tracking
- **Modern Architecture** - Containerized deployment with scalable microservices

## 🚀 Quick Start

Get Focus Mentor running in minutes with Docker!

```bash
# Clone the repository
git clone https://github.com/rishu685/focus-mentor-.git
cd focus-mentor-

# Copy environment template
cp .env.example .env

# Build and start with Docker Compose
docker-compose up --build -d

# Access the application
open http://localhost:3000
```

🎉 **That's it!** Your Focus Mentor instance is now running.

## 🔧 Installation Guide

### 📋 **Prerequisites**
- **Docker** (v20.10+) and **Docker Compose** (v2.0+) 
- **Node.js** (v18+) - *Optional, for local development*
- **MongoDB** (v6.0+) - *Optional, for local database*

### 🐳 **Docker Deployment (Recommended)**

<details>
<summary><b>🔧 Step-by-Step Docker Setup</b></summary>

1. **Clone and Navigate**
   ```bash
   git clone https://github.com/rishu685/focus-mentor-.git
   cd focus-mentor-
   ```

2. **Environment Configuration**
   ```bash
   # Copy the environment template
   cp .env.example .env
   
   # Edit environment variables (required for full functionality)
   nano .env  # or use your preferred editor
   ```

3. **Build and Deploy**
   ```bash
   # Build containers
   docker-compose build
   
   # Start services in detached mode
   docker-compose up -d
   
   # Check service status
   docker-compose ps
   ```

4. **Verify Installation**
   ```bash
   # Check application health
   curl http://localhost:3000/api/health
   
   # View logs
   docker-compose logs -f frontend
   docker-compose logs -f backend
   ```

5. **Access Services**
   - **Frontend**: http://localhost:3000
   - **Backend API**: http://localhost:8000
   - **MongoDB**: localhost:27017 (internal container network)
</details>

<details>
<summary><b>⚙️ Local Development Setup</b></summary>

For developers who prefer local development:

1. **Install Dependencies**
   ```bash
   # Install frontend dependencies
   npm install
   
   # Install backend dependencies
   cd server
   npm install
   cd ..
   ```

2. **Start MongoDB**
   ```bash
   # Using Docker
   docker run -d -p 27017:27017 --name focus-mentor-mongo mongo:6
   
   # Or use local MongoDB installation
   mongod --dbpath /usr/local/var/mongodb
   ```

3. **Development Servers**
   ```bash
   # Start backend (Terminal 1)
   cd server
   npm run dev
   
   # Start frontend (Terminal 2)
   npm run dev
   ```

4. **Access Development Environment**
   - **Frontend**: http://localhost:3000
   - **Backend**: http://localhost:8000
</details>

### 🐛 **Troubleshooting**

<details>
<summary><b>🔍 Common Issues & Solutions</b></summary>

**🐳 Docker Issues:**
```bash
# Permission denied (Linux/macOS)
sudo docker-compose up --build -d

# Port already in use
docker-compose down
lsof -ti:3000 | xargs kill -9  # Kill process on port 3000
docker-compose up -d

# Container build fails
docker system prune -a  # Clean Docker cache
docker-compose build --no-cache
```

**🔌 API Connection Issues:**
```bash
# Check backend health
curl http://localhost:8000/health

# Verify MongoDB connection
docker-compose exec backend npm run test:db

# Check environment variables
docker-compose exec frontend printenv | grep NEXT_PUBLIC
```

**🔐 Authentication Problems:**
```bash
# Regenerate NextAuth secret
openssl rand -base64 32

# Clear browser cookies and local storage
# Restart the application containers
docker-compose restart
```
</details>

### 🚀 **Getting Started**

1. **🍴 Fork the Repository**
   ```bash
   # Click the "Fork" button on GitHub
   git clone https://github.com/YOUR_USERNAME/focus-mentor-.git
   cd focus-mentor-
   ```

2. **🌿 Create a Feature Branch**
   ```bash
   git checkout -b feature/amazing-new-feature
   # or
   git checkout -b fix/important-bug-fix
   ```

3. **⚡ Set Up Development Environment**
   ```bash
   # Install dependencies
   npm install
   cd server && npm install && cd ..
   
   # Start development servers
   docker-compose up -d  # Database and services
   npm run dev          # Frontend
   cd server && npm run dev  # Backend
   ```

4. **✨ Make Your Changes**
   - Follow our [coding standards](CONTRIBUTING.md#coding-standards)
   - Write tests for new features
   - Update documentation as needed
   - Test across different browsers and devices

5. **📝 Commit Your Changes**
   ```bash
   git add .
   git commit -m "feat: add amazing new feature"
   # Follow conventional commit format
   ```

6. **🚀 Push and Create PR**
   ```bash
   git push origin feature/amazing-new-feature
   # Open a Pull Request on GitHub
   ```
